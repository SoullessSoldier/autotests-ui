# Практика работы с логированием
[назад](./readme.md)

В данном задании вам необходимо самостоятельно добавить логирование, используя функцию *get_logger*, к следующим компонентам, по аналогии с тем, как это было сделано в файле /elements/base_element.py в предыдущих шагах:
- **elements/button.py**: Добавьте логирование к методам *check_enabled* и *check_disabled*.
- **elements/input.py**: Добавьте логирование к методам *fill* и *check_have_value*.
- **elements/file_input.py**: Добавьте логирование к методу *set_input_files*.
- **elements/textarea.py**: Добавьте логирование к методам *fill* и *check_have_value*.
- **pages/base_page.py**: Добавьте логирование к методам *visit*, *reload* и *check_current_url*.
- **components/base_component.py**: Добавьте логирование к методу *check_current_url*.

## Критерии успешного выполнения
1. Логирование во всех указанных компонентах
2. Правильная инициализация логеров
- В **каждом файле** инициализирован **свой логгер** через get_logger("<NAME>").
- Имена логеров отражают назначение (например: "BUTTON", "INPUT", "BASE_PAGE", "BASE_COMPONENT" и т.п.).
3. Соответствие Allure шагам
- Все логи совпадают с текстом Allure шагов.
- Allure шаги вынесены в переменную *step*
4. Единый стиль сообщений
- Формат сообщений единообразен: *"Fill input "Login" to value "user"* / *"Checking that button "Submit" is enabled"*.
- Сообщения содержат: тип элемента / страницы, название, действие и значение (если применимо).
5. Запуск и проверка логов
- Все тесты запускаются и проходят:
    ```sh
    python -m pytest -m "regression" --alluredir=./allure-results
    ```
- В консоли отображаются логи (*logger.info*).
- В Allure-отчёте отображаются шаги, совпадающие с логами.
6. Код-стайл и аккуратность
- Нет дублирования текста шагов/логов (всегда используется *step* переменная).
- Код соответствует PEP8.
- Логика работы методов не изменена (добавлено только логирование).
